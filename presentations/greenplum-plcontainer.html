What is PL/Container

PL/Container is a customizable, secure runtime for Greenplum database procedural languages

  • Greenplum database extension
  • Stateless
  • Based on Docker containers
  • Customizable
  • Secure
  • Isolated

Untrusted -> Untrusted

PL/Container Architecture

PL/Container Architecture

Running UDFs in PL/Container

  1. PL/Container extension starts a docker container (only on first call)
  2. Transfer UDF and data to container
  3. Run the UDF in the container
  4. Contact the container to get the results

Install and Configure

Install PL/Container on Greenplum

Install from source

 https://github.com/greenplum-db/plcontainer 

git clone https://github.com/greenplum-db/plcontainer.git
cd plcontainer
make clean && make && make install

Install from GPPKG

 https://gpdb.docs.pivotal.io/6-1/ref_guide/extensions/pl_container.html 

gppkg -i plcontainer-2.0.2-gp6-rhel7_x86_64.gppkg
source $GPHOME/greenplum_path.sh
gpstop -ra

Run a simple Python function

CREATE LANGUAGE plpythonu;

CREATE OR REPLACE FUNCTION pylog10(input double precision)
RETURNS double precision AS $$
import math
return math.log10(input)
$$ LANGUAGE plpythonu;

SELECT pylog10(100);

Run a simple Python function

CREATE EXTENSION plcontainer;

CREATE OR REPLACE FUNCTION pylog10(input double precision)
RETURNS double precision AS $$
# container:plc_python_shared
import math
return math.log10(input)
$$ LANGUAGE plcontainer;

SELECT pylog10(100);
  pylog10
--------------
           2
(1 row)

PL/Container Performance

Long Running Function

CREATE OR REPLACE FUNCTION pysleep(i int) RETURNS void AS $$
# container: plc_python_shared
import time
time.sleep(i)
$$ LANGUAGE plcontainer;
SELECT count(pysleep(1)) FROM tbl;

PL/Container Performance

Large Input Array Function

CREATE OR REPLACE FUNCTION pylargeint8in(a int8[])
RETURNS float8 AS $$
#container : plc_python_shared
return sum(a)/float(len(a))
$$ LANGUAGE plcontainer;

SELECT count(pylargeint8in(ARRAY(SELECT column1 FROM tbl1)))
FROM tbl2;

2 times performance downgrade for Python
30% performance improvement for R

PL/Container Performance

Large Output Array Function

CREATE OR REPLACE FUNCTION pylargeoutfloat8(num int)
RETURNS float8[] AS $$
# container: plc_python_shared
return [x/3.0 for x in range(num)]
$$ LANGUAGE plcontainer;

SELECT count(pylargeoutfloat8(n)) FROM tbl;

4 times performance improvement
